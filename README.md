# محفّظ القرآن الكريم بالذكاء الاصطناعي — النموذج الأولي

تطبيق ويب يساعد على تثبيت الحفظ وتصحيح التلاوة: يستمع إليك وأنت تتلو،
ويتعرّف على صوتك، ويقارنه لحظةً بلحظة مع النص القرآني، ويُبرز موضعك على الشاشة،
ويلوّن الكلمة بالأحمر عند الاختلاف.

هذا النموذج الأولي مطبَّق على **سورة الفاتحة** (رواية حفص)، وبنيته قابلة لتوسعة
بقية السور بالطريقة نفسها.

> **المرحلة 1** (تتبّع لحظي، محاذاة مرنة، تلوين الأخطاء) +
> **المرحلة 2** (محرّك صوت محلي): التعرّف على التلاوة يجري الآن بنموذج
> **Whisper** يعمل **محلياً داخل المتصفّح** عبر Transformers.js — بلا خادم،
> ويعمل دون إنترنت بعد أول تحميل للنموذج.

---

## التشغيل

الميكروفون وعامل الويب يتطلّبان «سياقاً آمناً» (https أو `localhost`)، لذا شغّل
عبر الخادم المحلي بدل فتح الملف مباشرةً:

```bash
node server.js        # أو:  npm start
# ثم افتح:  http://localhost:8080
```

- اضغط **ابدأ الاستماع** وامنح إذن الميكروفون.
- عند أول مرّة يُحمَّل نموذج Whisper (~عشرات الميغابايت) ويُخزَّن في المتصفّح؛
  يظهر تقدّم التحميل بالنسبة المئوية، ثم يبدأ الاستماع.
- تظهر الكلمات المتعرَّف عليها عند نهاية كل وقفة/مقطع (يقطّع الصوت بكشف بسيط
  للنشاط الصوتي ثم يُمرّره إلى النموذج).
- لا تحتاج ميكروفوناً للتجربة: افتح **وضع الاختبار اليدوي** واكتب الكلمات.
- يعمل أفضل في **Chrome/Edge**. النشر على GitHub Pages يوفّر `https` المطلوب.

### الأداء

يستخدم النموذج عدّة تسريعات:

- **WebGPU:** يعمل الاستدلال على بطاقة الرسوميات عند توفّرها (Chrome/Edge)،
  وإلا يتراجع تلقائياً إلى WASM.
- **WASM متعدّد الخيوط:** عامل خدمة (`coi-serviceworker.js`) يضيف ترويسات
  COOP/COEP فيُمكّن العزل عبر الأصول و`SharedArrayBuffer`، فيعمل WASM على عدّة
  خيوط حتى على GitHub Pages (تُعاد الصفحة مرّة واحدة عند أول زيارة لتفعيله).
- **نموذج خفيف:** `Xenova/whisper-tiny` (تنزيل أصغر واستدلال أسرع).
- **تسخين مبدئي** بعد التحميل لتفادي بطء أوّل تعرّف.

يظهر المحرّك الفعلي (WebGPU أو WASM وعدد الخيوط) في سطر الملاحظة أسفل الصفحة.

## كيف يعمل

ثلاث طبقات مفصولة، كما في وثيقة المشروع:

1. **طبقة الصوت (ASR)** — `js/speech.js` + `js/whisper-worker.js`
   نموذج **Whisper** (`Xenova/whisper-base`) يعمل محلياً في المتصفّح عبر
   Transformers.js داخل **Web Worker** (حتى لا يتجمّد العرض أثناء التحميل أو
   الاستدلال). يلتقط `speech.js` الصوت من الميكروفون، ويقسّمه بكشف بسيط للنشاط
   الصوتي (VAD)، ويحوّله إلى 16 كيلوهرتز، ثم يرسله إلى العامل للتعرّف عليه.
   الطبقة معزولة خلف واجهة `start/stop/onWord`، فيسهل استبدال النموذج (مثلاً
   `whisper-tiny` أو محرّك آخر) دون المساس بمنطق المحاذاة.

2. **طبقة المحاذاة والتصحيح (الجوهر)** — `js/normalize.js` + `js/aligner.js`
   مستقلّة تماماً عن محرّك الصوت:
   - **التطبيع:** إزالة التشكيل والتطويل، وتوحيد الحروف المتقاربة
     (أ/إ/آ/ٱ ← ا ، ى ← ي ، ة ← ه …) لتفادي الأخطاء الكاذبة.
   - **المحاذاة:** لكل كلمة مسموعة نبحث في نافذة حول «المرساة»
     (الكلمة المتوقَّعة التالية) بمدى `[المرساة − 4 ، المرساة + 6]`، ونحسب
     تشابهاً عبر مسافة ليفنشتاين مع ترجيح القرب. إذا تجاوز أفضل تشابه
     **العتبة** (افتراضياً `0.62`) نثبّت الكلمة صحيحةً وننقل المرساة؛ وإلا
     نعلّم الموضع المتوقَّع خطأً مؤقتاً ونُبقي المرساة لإتاحة إعادة النطق.
     النافذة هي ما يسمح بالقفز للأمام والرجوع للخلف دون فقدان الموضع.

3. **طبقة العرض** — `index.html` + `css/styles.css` + `js/app.js`
   عرض من اليمين لليسار بخط مصحفي، وتلوين الكلمات: **أخضر** صحيحة،
   **ذهبي** الموضع الحالي، **أحمر** اختلاف، مع شريط تقدّم وإشعارات.

## بنية المشروع

```
index.html            الواجهة
css/styles.css         التنسيقات (RTL، خط مصحفي، ألوان الحالات)
js/quran-data.js       نص سورة الفاتحة (قابل لإضافة سور أخرى)
js/normalize.js        تطبيع النص العربي
js/aligner.js          محرّك المحاذاة (ليفنشتاين + نافذة بحث + عتبة)
js/speech.js           التقاط الصوت + VAD + إدارة عامل Whisper
js/whisper-worker.js   عامل ويب يشغّل Whisper عبر Transformers.js
js/app.js              ربط المكوّنات والعرض
test/aligner.test.js   اختبارات وحدة (Node، بلا تبعيات)
server.js              خادم تطوير محلي بسيط
```

أي نسخة لاحقة.
